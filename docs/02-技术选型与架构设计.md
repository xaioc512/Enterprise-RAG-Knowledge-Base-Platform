# 02 - 技术选型与架构设计

> 版本：v1.0 | 日期：2026-06-08

---

## 1. 技术选型

| 层次 | 技术 | 版本/说明 |
|------|------|-----------|
| 前端框架 | Vue 3 + Vite | Composition API，SPA模式 |
| UI组件库 | Element Plus | 企业级Vue 3组件库 |
| 状态管理 | Pinia | Vue 3 官方Store |
| HTTP客户端 | Axios | 请求拦截 + JWT自动注入 |
| 后端框架 | FastAPI | Python 3.10+，async/await |
| ASGI服务器 | Uvicorn + Gunicorn | 生产部署 |
| 依赖管理 | Poetry | 锁版本管理 |
| ORM | SQLAlchemy 2.0 | async模式 |
| 迁移工具 | Alembic | 数据库版本化迁移 |
| 异步队列 | Celery + Redis | 文档处理任务 |
| 认证 | python-jose + passlib | JWT + bcrypt |
| 日志 | Loguru | 结构化JSON日志 |
| AI框架 | LangChain + LangGraph | RAG Pipeline编排 |
| LLM | DeepSeek API | Chat + Embedding |
| 关系数据库 | MySQL 8.0 | 业务数据 |
| 缓存/队列 | Redis 7 | 会话缓存 + Celery Broker |
| 向量数据库 | Chroma | 文档向量存储与检索 |
| 文档解析 | PyPDF2 / python-docx / markdown | 多格式支持 |
| 反向代理 | Nginx | 静态资源 + API代理 |

## 2. 系统架构

```
用户浏览器 → Nginx(:80) → Vue3前端(静态)
                        → /api/* → FastAPI(:8000)
                                      ├── MySQL(:3306)
                                      ├── Redis(:6379)
                                      ├── Chroma(:8001)
                                      └── Celery Worker (独立进程)
```

## 3. 数据流

### RAG问答
```
用户提问 → JWT鉴权 → Embedding(问题) → Chroma检索(k=5)
→ 拼接上下文+对话历史 → DeepSeek Chat(流式) → SSE返回前端
```

### 文档入库（Celery异步）
```
管理员上传 → 保存文件 → 返回"处理中" → Celery任务入队
→ Worker: 解析 → 分块 → Embedding → Chroma写入 → 更新MySQL状态
```

## 4. 关键决策

- **为什么Celery而不是BackgroundTasks**：文档处理可能耗时数分钟，需要重试和状态追踪
- **为什么Chroma而不是Milvus**：轻量级，Python原生，适合中小规模知识库
- **为什么async SQLAlchemy**：与FastAPI async风格一致，避免阻塞事件循环
- **为什么SSE而不是WebSocket**：问答是单向流式输出，SSE更简单，Nginx兼容性更好
