## 2. 代码规范

### 命名
- 文件名：`snake_case.py`
- 类名：`PascalCase`
- 函数/变量：`snake_case`
- 常量：`UPPER_SNAKE_CASE`

### 分层原则
- **api/** — 仅负责参数校验、调用service、返回响应，不含业务逻辑
- **services/** — 全部业务逻辑，编排调用
- **models/** — 纯ORM定义，无业务方法
- **schemas/** — 纯Pydantic模型定义
- **rag/** — RAG专用模块，独立可测试

### 异步规范
- 所有数据库操作使用 `async session`
- 所有HTTP请求使用 `httpx.AsyncClient`
- Celery任务内部可用同步代码（独立进程）

## 3. 配置管理

使用 `pydantic-settings`，所有配置通过环境变量或 `.env` 文件注入：

```python
class Settings(BaseSettings):
    # 数据库
    MYSQL_HOST: str = "localhost"
    MYSQL_PORT: int = 3306
    MYSQL_USER: str
    MYSQL_PASSWORD: str
    MYSQL_DATABASE: str

    # Redis
    REDIS_URL: str = "redis://localhost:6379/0"

    # DeepSeek
    DEEPSEEK_API_KEY: str
    DEEPSEEK_BASE_URL: str = "https://api.deepseek.com"

    # JWT
    JWT_SECRET: str
    JWT_ALGORITHM: str = "HS256"
    JWT_EXPIRE_MINUTES: int = 1440  # 24h

    # Chroma
    CHROMA_PERSIST_DIR: str = "./chroma_data"

    model_config = SettingsConfigDict(env_file=".env")
```

## 4. 错误处理

- 统一异常处理：`@app.exception_handler` 全局注册
- 业务异常继承 `HTTPException`
- 所有异常记录Loguru日志

## 5. 日志规范

Loguru JSON格式输出，每条日志包含：
- `timestamp`: ISO 8601
- `level`: INFO/WARNING/ERROR/DEBUG
- `module`: 来源模块
- `message`: 日志内容
- `extra`: 请求ID、用户ID等上下文（可选）
